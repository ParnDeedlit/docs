GIS Agent can publish your spatial analysis results as live map services, shareable web applications, or downloadable files. This guide covers the publishing workflow from prompt to live service, tile type selection, API key management, and all supported export formats.

## Choosing the Right Output Type

<Tabs>
  <Tab title="Need interactive features?">
    If users need to click features, query attributes, or see dynamic labels, use a **Vector Tile Service**.

    ```
    Publish the result as a vector tile service.
    ```

    **Best for:** city maps, POI exploration, administrative boundaries — any layer where users need to interact with the data.
  </Tab>

  <Tab title="Need a high-performance base map?">
    If you need a fast-loading background layer — terrain, satellite imagery, or hillshade — use a **Raster Tile Service**.

    ```
    Publish the result as a raster tile service.
    ```

    **Best for:** terrain visualization, imagery base maps, and static analytical outputs where interactivity is not required.
  </Tab>

  <Tab title="Need a shareable web page?">
    For a complete, self-contained web application that non-GIS users can view in any browser, use an **HTML Application**.

    ```
    Generate an interactive HTML map page with layer controls and feature popups.
    ```

    **Best for:** stakeholder presentations, public portals, and embedded dashboards.
  </Tab>

  <Tab title="Need file export?">
    For reports, other GIS tools, or archiving, use file export.

    ```
    Export the current map as [PNG / GeoJSON / Shapefile / PDF / Word].
    ```

    See the [Export Formats Reference](#export-formats-reference) table below for the full list of supported formats.
  </Tab>
</Tabs>

## Publishing a Vector Tile Service

<Steps>
  <Step title="Complete your data preparation">
    Finish importing, clipping, and analyzing your spatial data before publishing. Verify that all layers display correctly in the preview panel.
  </Step>
  <Step title="Add the publishing instruction to your prompt">
    ```
    Publish the prepared data as a vector tile service.
    ```
  </Step>
  <Step title="Retrieve the tile service URL">
    GIS Agent publishes to the MapGIS spatiotemporal database and returns a tile service URL. Copy this URL for use in other applications.
  </Step>
  <Step title="View or share the result">
    View the map directly in the GIS Agent preview panel, share the URL with colleagues, or embed it in your GIS application.
  </Step>
</Steps>

<Note>
Tile service URLs generated by GIS Agent are accessible within your organization's network — or publicly, depending on your deployment configuration. Check with your system administrator if you are unsure which access policy applies.
</Note>

## Publishing a Raster Tile Service

For terrain and imagery data, publish as a raster tile service. The hillshade pattern below produces the best visual results for elevation data:

```text Hillshade raster tile
Generate a hillshade from the DEM data for [region], using azimuth 315°
and altitude 45°, then publish as a raster tile service for use as a base map.
```

<Tip>
Combining raster tiles with vector tile overlays is the recommended approach for production maps. Raster tiles load quickly as the background layer, while vector tiles provide interactive business data on top. Use the prompt `Combine the raster tile base map with the vector tile service as an overlay layer` to set this up.
</Tip>

## API Keys for Tile Services

<Note>
GIS Agent includes built-in API keys for services like Tianditu (天地图) for quick testing. These shared keys are rate-limited and should not be used in production applications. For production, apply for your own API key at the relevant provider portal, then use this prompt pattern:
</Note>

```text Custom API key injection
Use Tianditu as the base map with my API key: [your-api-key-here]
```

## Fixing Font Rendering Issues

<Note>
If map labels show garbled text or missing glyphs, the map may be missing required font files. Add the following to your prompt to load an external font and resolve the issue:
</Note>

```text Font fix
Use the font from https://fonts.loli.net/css?family=Open+Sans
```

## Sharing and Embedding

GIS Agent gives you three ways to share published maps:

1. **Direct URL**: Copy the service URL from the publishing result and share it with anyone who has access to your deployment.
2. **HTML embed**: Use the generated HTML page URL in an `<iframe>` tag to embed the map in another website or internal portal.
3. **WMS/WFS endpoint**: Connect the service URL to desktop GIS software such as QGIS or ArcGIS using the standard OGC protocol.

## Export Formats Reference

| Format | Prompt | Best For |
|---|---|---|
| PNG | `Export as PNG` | Reports, presentations |
| JPG | `Export as JPG` | Web sharing |
| GeoJSON | `Export as GeoJSON` | Sharing with developers or other GIS tools |
| Shapefile | `Export as Shapefile` | Desktop GIS (QGIS, ArcGIS) |
| PDF | `Export as PDF` | Print-ready analytical reports |
| Word | `Export as Word document` | Editable reports |
| Markdown | `Export as Markdown` | Technical documentation |
